# pgevolve v0.1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship pgevolve v0.1 — a Postgres-specific declarative schema-management CLI with full deploy → introspect → diff → plan → apply loop for tables, columns, indexes, constraints, and sequences.

**Architecture:** Cargo workspace of three crates (`pgevolve-core` library, `pgevolve` CLI binary, `pgevolve-testkit` dev infra). All downstream logic operates over a single normalized IR built from both source SQL (`pg_query.rs`) and live catalog (`pg_catalog`). Plans are directories of `plan.sql` + `intent.toml` + `manifest.toml`. Destructive changes require explicit per-plan approval. Audit lives in a minimal `pgevolve` schema in the target DB.

**Tech Stack:** Rust 1.78+, `pg_query` 5.x, `tokio-postgres`, `clap`, `serde` + `toml`, `tracing`, `thiserror`, `proptest`, `testcontainers`.

**Source spec:** [`docs/superpowers/specs/2026-05-09-pgevolve-design.md`](../specs/2026-05-09-pgevolve-design.md).

## Conventions used in every phase plan

- **File paths are absolute under the workspace root.** Workspace root is the directory containing the top-level `Cargo.toml`, i.e., the directory where `git init` ran.
- **Every code-bearing step shows the actual code.** No "implement X" without the body.
- **TDD is the default.** Each new public function or type lands as: write failing test → run to confirm fail → implement → run to confirm pass → commit. Some scaffolding tasks are exceptions (where there's no behavior to test, like adding a workspace member).
- **Commits are frequent and explanatory.** Conventional Commits style (`feat:`, `fix:`, `chore:`, `test:`, `docs:`, `refactor:`). One logical change per commit. Commit message ends with the standard `Co-Authored-By: Claude Opus 4.7 (1M context) <noreply@anthropic.com>` line.
- **Tier-1 and Tier-2 tests live next to the code** (`#[cfg(test)] mod tests` for unit tests; `tests/` directory for integration tests). Tier-3+ live in `pgevolve-testkit` and the per-crate `tests/` integration test trees.
- **No `unwrap()` / `expect()` in non-test code** except where a `// SAFETY:` style comment justifies it (e.g., a `Mutex::lock` that can't be poisoned by construction). Tests may unwrap freely.
- **Every public API has a doc comment.** `cargo doc` should be informative; `#![warn(missing_docs)]` is set on `pgevolve-core` and `pgevolve-testkit`.

## Self-review checkpoints

After each phase is implemented, before merging the phase branch:

1. **Spec coverage.** Does every spec section relevant to this phase have at least one task that implements it? Use the per-phase "spec coverage" map at the top of the phase file.
2. **Tier-1/2 test density.** Each new module should have ≥ 1 unit test for its happy path and ≥ 1 for each named error/edge case.
3. **No placeholder code.** Search for `todo!()`, `unimplemented!()`, `// TODO`, `// FIXME`. Resolve or convert to tracked GitHub issues.
4. **`cargo clippy --all-targets --all-features -- -D warnings`** passes.
5. **`cargo fmt --all -- --check`** passes.
6. **`cargo test --workspace`** passes (Tier 1 + 2 always; Tier 3+ where the harness exists).

## Glossary (anchors for the phase plans)

- **IR** — In-memory schema representation. The single shape both source and catalog produce. See [`phase-1-ir.md`](./phase-1-ir.md).
- **Source IR** / **Catalog IR** — Same Rust types; named by origin. They're interchangeable from the differ onward.
- **ChangeSet** — Ordered list of `Change` values produced by the differ. Pre-rewrite, pre-grouping.
- **Plan** — Post-rewrite, post-grouping, deterministic, hash-identified migration artifact.
- **PlanId** — `BLAKE3(canonical_serialize(source_ir) || canonical_serialize(catalog_ir) || pgevolve_version || planner_ruleset_version)`.
- **Step** — One executable SQL statement plus pgevolve metadata.
- **TransactionGroup** — Sequence of steps that share a single `BEGIN…COMMIT` (or, for `transactional=false`, run autocommit).
- **Destructiveness** — Tag on a Change/TableOp: `Safe`, `RequiresApproval { reason }`, or `RequiresApprovalAndDataLossWarning { reason }`. Drives `intent.toml` generation.
- **Managed schemas** — Schemas listed in `[managed].schemas` of `pgevolve.toml`. The diff engine sees only objects within these schemas, minus `ignore_objects`, minus the `pgevolve` metadata schema (always implicitly excluded).
- **Tiers 1–7** — Test categories from spec §14: unit, parser fixture corpus, catalog round-trip golden, end-to-end migration fixture, property-based, multi-version matrix, soak.
