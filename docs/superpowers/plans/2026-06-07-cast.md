# CREATE CAST Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the global `CAST` object kind (`WITH FUNCTION` / `WITHOUT FUNCTION` / `WITH INOUT`, contexts `EXPLICIT`/`ASSIGNMENT`/`IMPLICIT`), with `WITH FUNCTION` constrained to managed SQL/plpgsql functions.

**Architecture:** New `Cast` IR on `Catalog`, parsed from `CreateCastStmt`, read from `pg_cast`⋈`pg_type`×2⋈`pg_proc` (excluding system `oid<16384` and extension-owned casts), diffed as a **managed global** object (Create/Drop/Replace/CommentOn — no owner, no `ALTER CAST`), rendered with new `StepKind`s, with a closed-world constraint that `WITH FUNCTION` resolves to a managed function and dep-graph edges to that function and to managed source/target types.

**Tech Stack:** Rust, `pg_query`, `pg_catalog` introspection, conformance harness.

**Design:** [`docs/superpowers/specs/2026-06-07-cast-design.md`](../specs/2026-06-07-cast-design.md)

**Closest template:** the just-shipped **AGGREGATE** feature ([`2026-06-06-aggregate.md`](2026-06-06-aggregate.md)) — CAST is structurally identical (global object, managed drop, managed-function constraint, closed-world lint, dep edges to a function) but **simpler**: no owner, simpler method/context enums, and identity is a `(QualifiedName, QualifiedName)` type pair instead of `(qname, arg_types)`. For every "mirror X" below, the AGGREGATE commit for that layer is the reference.

## Verified facts
- `CREATE CAST` is a pg_query `CreateCastStmt` with fields `sourcetype: TypeName`, `targettype: TypeName`, `func: Option<ObjectWithArgs>`, `context: CoercionContext` (`CoercionExplicit=4`, `CoercionAssignment=2`, `CoercionImplicit=1`), `inout: bool`. Method: `func.is_some()` → Function; else `inout` → Inout; else → Binary.
- `NodeId::Type(QualifiedName)` (`plan/edges.rs:56`) is the user-type node for source/target edges; `NodeId::Function(QualifiedName, NormalizedArgTypes)` (`:58`) is the cast-function node — mirror the aggregate→function edge.
- The aggregate closed-world lint is `lint/rules/aggregate_unmanaged_state_function.rs`, wired in `lint/universal.rs::check_universal` (`:157`) and emits a plan-time `Finding::error("aggregate-references-unmanaged-function", …)`. Mirror it.
- Reader: resolve type strings via `ColumnType::parse_from_pg_type_string` (`catalog/assemble/aggregates.rs:119`); the aggregate reader excludes extension-owned objects via `pg_depend deptype='e'` — reuse that join.
- `pg_cast` has **no owner column** → no `AlterOwner` path anywhere.
